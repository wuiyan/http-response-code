## 定义

```
HTTP 429 Too Many Requests 表示用户在给定的时间内发送了太多请求，超出了服务器的限制。
```

## 描述

HTTP 429 状态码用于速率限制，当客户端发送的请求过于频繁，超出服务器设定的限制时返回。

常见使用场景：
- API 限流保护
- 防止暴力破解攻击
- 防止爬虫过度抓取
- 保护服务器资源不被耗尽

服务器通常会在响应中包含 Retry-After 头，告知客户端何时可以重新发送请求。客户端收到此状态码后应暂停发送请求，等待指定时间后重试。
